⏺ Both clean. The fix was two lines:

  tsconfig.json — before:
  {
    "extends": "expo/tsconfig.base",
    "compilerOptions": {
      "strict": true,
      "types": ["jest"]       ← didn't work reliably on CI
    },
    "exclude": ["node_modules"]
  }

  tsconfig.json — after:
  {
    "extends": "expo/tsconfig.base",
    "compilerOptions": {
      "strict": true
    },
    "exclude": ["node_modules", "__tests__", "__mocks__"]  ← exclude test files
  entirely
  }

  Why this works: expo/tsconfig.base manages its own types array internally.
  Trying to append jest to it via extension isn't guaranteed to merge — it can
  silently replace the whole array or conflict, depending on the TypeScript
  version on the runner. The reliable solution is to simply exclude __tests__/
  and __mocks__/ from tsc's scope. tsc --noEmit now only checks your source
  code. Jest's own transformer (via jest-expo) type-checks test files separately
   when tests run, so nothing is left unchecked.
